PROBLEM STATEMENT

There are numCourses courses labeled 0 to numCourses-1.

Given prerequisites where prerequisites[i] = [ai, bi] means
you MUST take course bi BEFORE course ai.
  → [ai, bi] means: bi → ai  (same format as Problem 21)

Return a valid ORDER to take all courses.
If impossible (cycle exists), return an EMPTY array.

Example 1:
  numCourses = 4
  prerequisites = [[1,0], [2,0], [3,1], [3,2]]
  
  Output: [0, 1, 2, 3]  or  [0, 2, 1, 3]
  (multiple valid answers — return ANY one)

Example 2:
  numCourses = 2
  prerequisites = [[1,0], [0,1]]
  
  Output: []  (cycle — impossible)

Example 3:
  numCourses = 1
  prerequisites = []
  
  Output: [0]

Solution 1: Kahn's Algorithm (BFS) — Recommended for Interviews

from collections import deque

def findOrder_kahn(numCourses, prerequisites):
    """
    Return a valid course ordering, or [] if impossible.
    
    THIS IS: Problem 19 (Kahn's) + Problem 21 (edge format + cycle check)
    
    Combines:
      → Graph building with [course, prereq] format (Problem 21)
      → Full Kahn's algorithm returning result list (Problem 19)
      → Cycle detection via length check (Problem 19/21)
    """
    # ─── BUILD GRAPH (same edge parsing as Problem 21) ───
    graph = [[] for _ in range(numCourses)]
    in_degree = [0] * numCourses
    
    for course, prereq in prerequisites:
        graph[prereq].append(course)    # prereq → course
        in_degree[course] += 1
    
    # ─── KAHN'S BFS (same as Problem 19) ───
    queue = deque(i for i in range(numCourses) if in_degree[i] == 0)
    result = []
    
    while queue:
        node = queue.popleft()
        result.append(node)             # ← Problem 21 used count; here we store
        
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # ─── CYCLE CHECK + RETURN ───
    if len(result) == numCourses:
        return result                   # valid topological order
    else:
        return []                       # cycle exists


# ─── TESTS ───
print(findOrder_kahn(4, [[1,0],[2,0],[3,1],[3,2]]))   # [0, 1, 2, 3] or [0, 2, 1, 3]
print(findOrder_kahn(2, [[1,0],[0,1]]))                 # [] (cycle)
print(findOrder_kahn(1, []))                            # [0]
print(findOrder_kahn(4, []))                            # [0, 1, 2, 3] (any order)

Solution 2: DFS Reverse Post-Order

from collections import defaultdict

def findOrder_dfs(numCourses, prerequisites):
    """
    Return valid course ordering using DFS approach.
    
    THIS IS: Problem 20 (DFS topo) + Problem 21 (edge format + cycle check)
    
    Combines:
      → Three-state DFS cycle detection (Problem 11/20)
      → Post-order collection (Problem 20)
      → Edge format from Problem 21
    """
    # ─── BUILD GRAPH ───
    graph = defaultdict(list)
    for course, prereq in prerequisites:
        graph[prereq].append(course)
    
    # ─── THREE-STATE DFS (from Problem 20) ───
    WHITE, GRAY, BLACK = 0, 1, 2
    state = [WHITE] * numCourses
    post_order = []
    has_cycle = False
    
    def dfs(node):
        nonlocal has_cycle
        if has_cycle:
            return
        
        state[node] = GRAY
        
        for neighbor in graph[node]:
            if state[neighbor] == GRAY:
                has_cycle = True        # back edge → cycle (Problem 11)
                return
            if state[neighbor] == WHITE:
                dfs(neighbor)
                if has_cycle:
                    return
        
        state[node] = BLACK
        post_order.append(node)         # ← record finish order (Problem 20)
    
    # ─── VISIT ALL NODES ───
    for node in range(numCourses):
        if state[node] == WHITE:
            dfs(node)
            if has_cycle:
                return []
    
    # ─── REVERSE POST-ORDER = TOPOLOGICAL ORDER (Problem 20) ───
    post_order.reverse()
    return post_order


# ─── TESTS ───
print(findOrder_dfs(4, [[1,0],[2,0],[3,1],[3,2]]))    # [0, 2, 1, 3] or similar
print(findOrder_dfs(2, [[1,0],[0,1]]))                  # []
print(findOrder_dfs(1, []))                             # [0]

Solution 3: DFS Iterative (No Recursion Limit)

from collections import defaultdict, deque

def findOrder_iterative(numCourses, prerequisites):
    """
    Iterative DFS — safe for large inputs.
    
    Uses (node, processed) stack trick from Problem 20 Version 3.
    """
    graph = defaultdict(list)
    for course, prereq in prerequisites:
        graph[prereq].append(course)
    
    WHITE, GRAY, BLACK = 0, 1, 2
    state = [WHITE] * numCourses
    post_order = []
    
    for start in range(numCourses):
        if state[start] != WHITE:
            continue
        
        stack = [(start, False)]
        
        while stack:
            node, processed = stack.pop()
            
            if processed:
                # Node finishing → add to post-order
                state[node] = BLACK
                post_order.append(node)
                continue
            
            if state[node] != WHITE:
                continue    # already in-progress or done
            
            state[node] = GRAY
            stack.append((node, True))      # push finish marker
            
            for neighbor in graph[node]:
                if state[neighbor] == GRAY:
                    return []               # cycle!
                if state[neighbor] == WHITE:
                    stack.append((neighbor, False))
    
    post_order.reverse()
    return post_order


# ─── TEST ───
print(findOrder_iterative(4, [[1,0],[2,0],[3,1],[3,2]]))
print(findOrder_iterative(2, [[1,0],[0,1]]))

Solution 4: Kahn's with Layer Tracking (Bonus — For Parallel Scheduling)

from collections import deque

def findOrder_with_semesters(numCourses, prerequisites):
    """
    BONUS: Not only find the order, but also group by SEMESTERS.
    
    Each BFS layer = one semester (courses that can be taken simultaneously).
    This showcases Kahn's UNIQUE advantage over DFS (Problem 19 note).
    
    Returns: (order, semesters)
      order = flat topological order
      semesters = list of lists, each inner list = one semester
    """
    graph = [[] for _ in range(numCourses)]
    in_degree = [0] * numCourses
    
    for course, prereq in prerequisites:
        graph[prereq].append(course)
        in_degree[course] += 1
    
    queue = deque(i for i in range(numCourses) if in_degree[i] == 0)
    result = []
    semesters = []
    
    while queue:
        # Process entire current level (one semester)
        # Same level-by-level pattern as Problem 13 (Rotten Oranges)
        semester = []
        level_size = len(queue)
        
        for _ in range(level_size):
            node = queue.popleft()
            result.append(node)
            semester.append(node)
            
            for neighbor in graph[node]:
                in_degree[neighbor] -= 1
                if in_degree[neighbor] == 0:
                    queue.append(neighbor)
        
        semesters.append(semester)
    
    if len(result) == numCourses:
        return result, semesters
    else:
        return [], []


# ─── TEST ───
order, semesters = findOrder_with_semesters(6, [[1,0],[2,0],[3,1],[3,2],[4,3],[5,1]])

print(f"Order: {order}")
print(f"Semesters ({len(semesters)} total):")
for i, sem in enumerate(semesters):
    print(f"  Semester {i+1}: {sem}")

STEP 7: WHAT IF THE GRAPH WERE...

What if We Need Lexicographically Smallest Order?

  "Among all valid orderings, return the lexicographically smallest"
  
  Example:
    Both [0, 1, 2, 3] and [0, 2, 1, 3] are valid.
    Lexicographically smallest: [0, 1, 2, 3]  (1 < 2 at position 1)
  
  SOLUTION: Replace deque with min-heap in Kahn's (mentioned in Problem 19)
  
  import heapq
  
  heap = [i for i in range(numCourses) if in_degree[i] == 0]
  heapq.heapify(heap)
  
  while heap:
      node = heapq.heappop(heap)  # always pick SMALLEST available
      result.append(node)
      for neighbor in graph[node]:
          in_degree[neighbor] -= 1
          if in_degree[neighbor] == 0:
              heapq.heappush(heap, neighbor)
  
  Time: O((V + E) log V) — heap operations add log V factor
  
  NOTE: DFS approach does NOT naturally give lexicographic order.
  This is a case where Kahn's clearly wins.

What if We Need the Minimum Number of Semesters?

  "How many semesters do you need if you can take unlimited courses per semester?"
  
  → This is the number of BFS LAYERS in Kahn's (Solution 4)
  → = length of longest path in the DAG
  → = critical path in project scheduling
  
  Kahn's with layer tracking gives this for FREE.
  DFS would need separate computation.
  
  Another case where Kahn's wins over DFS.

What if There's a Limit of K Courses Per Semester?

  "You can take at most K courses per semester. Minimum semesters?"
  
  → Kahn's layers tell you what CAN be taken simultaneously
  → But if a layer has more than K courses, split across semesters
  → Greedy: take K courses from available pool each semester
  → Still based on Kahn's but with capacity constraint
  → More complex problem — beyond basic topo sort

What if Prerequisites Have Weights (Time to Complete)?

  "Each course takes X weeks. Minimum total time with parallelism?"
  
  → This becomes "Critical Path Method" (CPM)
  → Topological sort + DP on DAG (Problem 47 preview)
  → Process in topo order, compute earliest completion time per course
  → Answer = max completion time among all courses
