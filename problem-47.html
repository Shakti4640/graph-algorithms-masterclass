[Algorithm Family: Topological Sort + Dynamic Programming]

PROBLEM STATEMENT

You are managing a project with tasks and dependencies.
Each task takes a certain number of days to complete.
Some tasks depend on others (must finish before starting the next).

Find the MINIMUM number of days to complete the ENTIRE project.
(Tasks without dependencies between them can run in PARALLEL.)

This equals the LONGEST PATH through the dependency graph,
because the bottleneck is the longest chain of sequential tasks.

Input:
  n = 8  (tasks numbered 0 to 7)
  durations = [3, 2, 4, 1, 5, 2, 3, 1]
  dependencies = [[0,2], [0,3], [1,3], [2,4], [2,5], [3,5], [4,6], [5,6], [6,7]]
  (dependency [a,b] means task a must finish before task b starts)

Output:
  Minimum days to finish all tasks = ?
  (= longest weighted path from any start to any end)

Constraints:
  → Up to 100,000 tasks
  → Dependencies form a DAG (no circular dependencies)
  → Need O(V + E) solution

Why "Longest Path" = "Minimum Project Duration"

  ═══════════════════════════════════════════════════════════════
  KEY INSIGHT: Critical Path Method (CPM)
  ═══════════════════════════════════════════════════════════════
  
  Tasks WITHOUT dependencies between them run in PARALLEL.
  The bottleneck is the longest SEQUENTIAL chain.
  
  Example chains:
    Chain A: 0(3) → 2(4) → 4(5) → 6(3) → 7(1)  = 3+4+5+3+1 = 16
    Chain B: 0(3) → 2(4) → 5(2) → 6(3) → 7(1)  = 3+4+2+3+1 = 13
    Chain C: 0(3) → 3(1) → 5(2) → 6(3) → 7(1)  = 3+1+2+3+1 = 10
    Chain D: 1(2) → 3(1) → 5(2) → 6(3) → 7(1)  = 2+1+2+3+1 = 9
    
  Longest chain = 16 (Chain A)
  → Project takes at least 16 days
  → This is the "critical path"
  
  WHY? Even though chains B, C, D finish earlier,
  chain A hasn't finished yet. Task 7 can't start until
  task 6 finishes, which can't start until task 4 finishes...
  
  The project is only done when ALL tasks are done,
  and the longest chain dictates the minimum time.

STEP 2: BRUTE FORCE FIRST

Approach: Enumerate All Paths

  Find every possible path from any source to any sink.
  Compute the weighted length of each.
  Return the maximum.
  
  Number of paths in a DAG can be EXPONENTIAL!
  
  Consider a DAG shaped like this:
  
    0 → 1 → 2 → 3 → ... → N-1
    0 → 2, 0 → 3, ..., 0 → N-1
    1 → 3, 1 → 4, ..., 1 → N-1
    ...
    
  Number of paths: O(2^N) in worst case
  
  For N = 100,000: IMPOSSIBLE

Why Brute Force Fails

  Enumerating all paths: O(2^N) — exponential
  Even smarter brute force (DFS from each source): O(V × (V+E))
  
  We need: O(V + E) — single pass through the graph
  
  KEY REALIZATION:
    → DAG has a topological order (Problems 19-20)
    → In topological order, ALL predecessors processed BEFORE current node
    → This is EXACTLY what DP needs: "subproblems solved before current"
    → Topological order IS the DP evaluation order for DAGs!

STEP 3: IDENTIFYING THE PATTERN

The Connection: Tree DP → DAG DP

  ═══════════════════════════════════════════════════════════════
  Problem 46 (Tree DP):
    → Tree is a special DAG (rooted: parent → child)
    → DFS post-order = process children before parent
    → At each node: combine children's answers
    → O(N) single pass
    
  Problem 47 (DAG DP):
    → General DAG (not a tree — multiple predecessors possible!)
    → Topological sort = process predecessors before current
    → At each node: combine predecessors' answers
    → O(V + E) single pass
    
  THE GENERALIZATION:
    Tree DP uses DFS post-order as the "topological sort"
    DAG DP uses actual topological sort (Kahn's BFS from Problem 19)
    
  WHY trees don't need explicit topo sort:
    → Each node has exactly ONE parent
    → DFS naturally processes children before parent
    
  WHY general DAGs need explicit topo sort:
    → A node can have MULTIPLE predecessors
    → Must ensure ALL predecessors are processed first
    → Kahn's algorithm (Problem 19) handles this with in-degree tracking
  ═══════════════════════════════════════════════════════════════

The DP Formulation

  STATE:
    dp[node] = longest path ending at node
             = earliest FINISH time of this task
             
  TRANSITION:
    dp[node] = duration[node] + max(dp[predecessor] for all predecessors)
    
    "I can't start until ALL my prerequisites finish.
     The latest-finishing prerequisite determines my start time.
     Then I add my own duration."
     
  BASE CASE:
    For source nodes (no predecessors / in-degree = 0):
    dp[source] = duration[source]
    "I can start immediately, so I finish at time = my duration."
    
  ANSWER:
    max(dp[node] for all nodes)
    "Project finishes when the LAST task finishes."
    
  EVALUATION ORDER:
    Topological order — guarantees all predecessors of node X
    are processed before X.

Why Topological Order is Perfect for DP

  ═══════════════════════════════════════════════════════════════
  THE FUNDAMENTAL INSIGHT
  ═══════════════════════════════════════════════════════════════
  
  DP requires: "When I compute dp[X], all dp[Y] that X depends on 
                must already be computed."
                
  In a DAG with edge Y → X:
    → X depends on Y (Y must be done before X)
    → dp[X] uses dp[Y]
    → So Y must be computed before X
    
  Topological sort guarantees EXACTLY this:
    → If edge Y → X exists, Y appears before X in the order
    → So when we reach X, dp[Y] is already computed
    
  This is WHY DP on DAGs works:
    "Topological sort gives us a valid DP evaluation order."
    
  And this is WHY DP on cyclic graphs DOESN'T work (in general):
    "No topological sort exists → no valid evaluation order."
    "Node A depends on B which depends on C which depends on A."
    "Chicken-and-egg problem → can't compute any of them first."

from collections import defaultdict, deque

def longest_path_dag(n, edges, durations):
    """
    Find the longest weighted path in a DAG with node weights.
    
    Equivalent to: minimum project completion time with dependencies.
    Also known as: Critical Path Method (CPM).
    
    Algorithm: Kahn's Topological Sort (Problem 19) + DP
    
    dp[node] = earliest finish time of this node
             = duration[node] + max(dp[predecessor] for all predecessors)
    
    Evaluation order: topological (Kahn's BFS)
    → Guarantees all predecessors computed before current node
    
    Built on:
      - Problem 1:  Adjacency list
      - Problem 19: Kahn's topological sort (BFS with in-degrees)
      - Problem 46: Tree DP (same bottom-up idea, but tree-specific)
    
    Time:  O(V + E)
    Space: O(V + E)
    """
    # ─── Build adjacency list and in-degrees (Problems 1, 19) ───
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
    
    # ─── Initialize DP and queue ───
    # dp[node] starts at 0 (earliest start = 0, will add duration when processed)
    # Actually dp[node] accumulates max finish time of predecessors
    dp = [0] * n
    
    # Enqueue all source nodes (in-degree = 0)
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            queue.append(node)
    
    # ─── Kahn's BFS + DP ───
    topo_order = []  # optional: store for debugging/tracing
    
    while queue:
        node = queue.popleft()
        topo_order.append(node)
        
        # dp[node] currently = max finish time of all predecessors
        # Add this node's duration to get THIS node's finish time
        dp[node] += durations[node]
        
        # Propagate to successors
        for neighbor in adj[node]:
            # neighbor can't start until THIS node finishes
            dp[neighbor] = max(dp[neighbor], dp[node])
            
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # ─── Check for cycle (Problem 21) ───
    if len(topo_order) != n:
        return -1  # cycle detected, not a valid DAG
    
    # ─── Answer = max finish time across all nodes ───
    return max(dp)

def longest_path_edge_weighted(n, weighted_edges):
    """
    Longest path in a DAG with EDGE weights (not node weights).
    
    Common in: shortest/longest distance problems on DAGs.
    
    dp[node] = longest distance from ANY source to this node
    dp[node] = max(dp[pred] + weight(pred, node)) for all predecessors
    
    Base: dp[source] = 0 (sources have distance 0 from themselves)
    Answer: max(dp)
    """

    for u, v, w in weighted_edges:
        adj[u].append((v, w))
        in_degree[v] += 1

        node = queue.popleft()
        
        for neighbor, weight in adj[node]:
            # Relax: can we reach neighbor with a LONGER path through node?
            dp[neighbor] = max(dp[neighbor], dp[node] + weight)
            
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    return max(dp)

from collections import defaultdict, deque

def shortest_path_dag(n, weighted_edges, source):
    """
    SHORTEST path from source in a DAG.
    
    Why not Dijkstra? Dijkstra works but is O((V+E) log V).
    DAG topo sort + DP is O(V + E) — FASTER!
    
    Also handles NEGATIVE weights (Dijkstra can't — Problem 23).
    Also handles negative weights better than Bellman-Ford O(VE) — Problem 26.
    
    dp[node] = shortest distance from source to node
    dp[node] = min(dp[pred] + weight(pred, node))
    
    KEY INSIGHT: for DAGs, topo sort + relaxation beats ALL 
    other shortest path algorithms in time complexity.
    
    ┌─────────────────┬────────────┬──────────────────┐
    │ Algorithm        │ Time       │ Handles negatives │
    ├─────────────────┼────────────┼──────────────────┤
    │ BFS (Problem 9)  │ O(V+E)     │ Unweighted only  │
    │ Dijkstra (P23)   │ O((V+E)lgV)│ No negatives     │
    │ Bellman-Ford(P26)│ O(VE)      │ Yes              │
    │ DAG DP (THIS)    │ O(V+E)     │ Yes (DAG only!)  │
    └─────────────────┴────────────┴──────────────────┘
    """
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v, w in weighted_edges:
        adj[u].append((v, w))
        in_degree[v] += 1
    
    # Initialize distances to infinity
    dist = [float('inf')] * n
    dist[source] = 0
    
    # Topological sort first (Kahn's)
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            queue.append(node)
    
    topo_order = []
    while queue:
        node = queue.popleft()
        topo_order.append(node)
        for nei, w in adj[node]:
            in_degree[nei] -= 1
            if in_degree[nei] == 0:
                queue.append(nei)
    
    # DP relaxation in topological order
    for node in topo_order:
        if dist[node] == float('inf'):
            continue  # unreachable from source
        for nei, weight in adj[node]:
            if dist[node] + weight < dist[nei]:
                dist[nei] = dist[node] + weight
    
    return dist

from collections import defaultdict, deque

def longest_path_with_reconstruction(n, edges, durations):
    """
    Find longest path AND reconstruct which nodes are on it.
    
    Extension: track predecessor for each node in the DP.
    When dp[neighbor] is updated, record that 'node' is 
    the predecessor on the longest path to 'neighbor'.
    
    Then backtrack from the node with maximum dp value.
    """
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
    
    dp = [0] * n
    predecessor = [-1] * n  # track which node led to best dp
    
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            queue.append(node)
    
    topo_order = []
    
    while queue:
        node = queue.popleft()
        topo_order.append(node)
        
        dp[node] += durations[node]
        
        for neighbor in adj[node]:
            if dp[node] > dp[neighbor]:
                dp[neighbor] = dp[node]
                predecessor[neighbor] = node  # node led to best path for neighbor
            
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # Find the end of the longest path
    end_node = max(range(n), key=lambda x: dp[x])
    longest = dp[end_node]
    
    # Reconstruct path by backtracking
    path = []
    node = end_node
    while node != -1:
        path.append(node)
        node = predecessor[node]
    path.reverse()
    
    return longest, path

from collections import defaultdict, deque

def count_longest_paths(n, edges, durations):
    """
    How many distinct critical paths exist?
    
    Track count[node] = number of longest paths ending at node.
    
    When we update dp[neighbor]:
      - If dp[node] > dp[neighbor]: new best → count[neighbor] = count[node]
      - If dp[node] == dp[neighbor]: equally good → count[neighbor] += count[node]
    """
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
    
    dp = [0] * n
    count = [0] * n
    
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            queue.append(node)
            count[node] = 1  # one path (just this source)
    
    while queue:
        node = queue.popleft()
        dp[node] += durations[node]
        
        for neighbor in adj[node]:
            if dp[node] > dp[neighbor]:
                dp[neighbor] = dp[node]
                count[neighbor] = count[node]  # reset: new best
            elif dp[node] == dp[neighbor]:
                count[neighbor] += count[node]  # add: equally good
            
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # Answer: count of longest paths to the overall endpoint
    max_dp = max(dp)
    total = sum(count[i] for i in range(n) if dp[i] == max_dp)
    
    return max_dp, total

Version 6: DFS-Based Approach (Alternative to Kahn's)

from collections import defaultdict

def longest_path_dfs(n, edges, durations):
    """
    Alternative: DFS with memoization instead of Kahn's BFS.
    
    Connects to Problem 20 (DFS topological sort).
    
    dp[node] = longest path starting FROM this node
    dp[node] = duration[node] + max(dp[successor] for all successors)
    
    NOTE: This computes longest path FROM each node (forward),
    whereas the Kahn's approach computes longest path TO each node.
    Both give the same final answer.
    
    Memoization ensures each node computed only once → O(V + E).
    """
    adj = defaultdict(list)
    for u, v in edges:
        adj[u].append(v)
    
    dp = [-1] * n  # -1 means "not computed yet"
    
    def dfs(node):
        """Returns longest path starting from this node."""
        if dp[node] != -1:
            return dp[node]  # already computed (memoization)
        
        # Base case: sink node (no successors)
        best_successor = 0
        
        for neighbor in adj[node]:
            best_successor = max(best_successor, dfs(neighbor))
        
        dp[node] = durations[node] + best_successor
        return dp[node]
    
    # Compute for all nodes
    for node in range(n):
        if dp[node] == -1:
            dfs(node)
    
    return max(dp)

  STRUCTURAL COMPARISON
  ═══════════════════════════════════════════════════════════════
  
  TREE:                          DAG:

  ┌──────────────────────┬────────────────────┬──────────────────┐
  │  Aspect              │  Tree DP (P46)     │  DAG DP (P47)    │
  ├──────────────────────┼────────────────────┼──────────────────┤
  │  Evaluation order    │  DFS post-order    │  Topological sort │
  │  Why it works        │  Children before   │  Predecessors     │
  │                      │  parent (natural)  │  before current   │
  │  Implementation      │  Recursive DFS     │  Kahn's BFS + DP │
  │  Multiple parents?   │  NO (tree)         │  YES (DAG)        │
  │  Combination rule    │  Combine children  │  max/min over     │
  │                      │  (top-2, etc.)     │  predecessors     │
  │  Path structure      │  Unique (tree path)│  Multiple paths   │
  │  Return vs Record    │  Different (P46)   │  Usually same     │
  │  Time complexity     │  O(N)              │  O(V + E)         │
  └──────────────────────┴────────────────────┴──────────────────┘
  
  KEY DIFFERENCE:
  
  Tree DP (Problem 46): path "bends" at turning point
    → Return chain (one direction) to parent
    → Record bent path (two directions) globally
    → Return ≠ Record
    
  DAG DP (Problem 47): path is always "forward" (follows edges)
    → dp[node] = best path TO this node
    → No "bending" — just accumulate along edges
    → Return = Record (simpler!)
    
  WHY Tree DP is trickier:
    In a tree, a "path" can go up and then down (bends).
    In a DAG, a "path" only follows edge directions (no bending).

from collections import defaultdict, deque

def count_paths_dag(n, edges, source, target):
    """
    Count the number of distinct paths from source to target in a DAG.
    
    dp[node] = number of paths from source to node
    dp[source] = 1
    dp[node] = sum(dp[predecessor] for all predecessors with edge to node)
    
    Why this works:
      Each predecessor contributes its own set of paths.
      All paths through pred_A are distinct from paths through pred_B.
      So we SUM (not max).
    """
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
    
    dp = [0] * n
    dp[source] = 1  # one path: just the source itself
    
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            queue.append(node)
    
    while queue:
        node = queue.popleft()
        for nei in adj[node]:
            dp[nei] += dp[node]  # SUM, not max
            in_degree[nei] -= 1
            if in_degree[nei] == 0:
                queue.append(nei)
    
    return dp[target]

def lis_dag(nums):
    """
    Longest Increasing Subsequence via DAG DP.
    
    Model as DAG:
      → Node i represents nums[i]
      → Edge i → j exists if i < j AND nums[i] < nums[j]
      → LIS = longest path in this DAG + 1
    
    This is O(N²) — not the fastest LIS (O(N log N) with patience sort)
    but it DEMONSTRATES DAG DP thinking beautifully.
    
    Connection to Problem 47:
      The DP transition dp[j] = 1 + max(dp[i] for i < j if nums[i] < nums[j])
      is EXACTLY longest path in DAG!
      The "natural ordering" i < j is a topological order.
    """
    n = len(nums)
    dp = [1] * n  # each element is a subsequence of length 1
    
    # Process in topological order (0, 1, 2, ..., n-1)
    # This IS topological because edges only go from smaller to larger index
    for j in range(1, n):
        for i in range(j):
            if nums[i] < nums[j]:
                dp[j] = max(dp[j], dp[i] + 1)
    
    return max(dp)

def coin_change_ways(coins, target):
    """
    Count ways to make 'target' using given coins (unlimited supply).
    
    This is DP, but CAN be viewed as DAG DP:
      → Nodes: amounts 0, 1, 2, ..., target
      → Edge from amount A to amount A+coin for each coin
      → Count paths from 0 to target = number of ways
    
    Shows that MANY classical DP problems are secretly DAG DP!
    """
    dp = [0] * (target + 1)
    dp[0] = 1  # one way to make amount 0: use no coins
    
    for coin in coins:
        for amount in range(coin, target + 1):
            dp[amount] += dp[amount - coin]
    
    return dp[target]

  │  Edit Distance         │  Node (i,j) → (i-1,j), (i,j-1),    │
  │                        │               (i-1,j-1)              │
  │                        │  Topo order: row by row              │
  ├────────────────────────┼──────────────────────────────────────┤
  │  Shortest path in DAG  │  Node u → all successors v           │

  ├────────────────────────┼──────────────────────────────────────┤
  │  Tree DP (Problem 46)  │  Node u → all children of u          │

  └────────────────────────┴──────────────────────────────────────┘

  WHY THIS MATTERS
  ═══════════════════════════════════════════════════════════════
  
  1. If you can MODEL a problem as "find optimal path/value in a DAG,"
     you've found a DP solution.
     
  2. The evaluation order of DP IS the topological order of the 
     dependency DAG.

  LONGEST  path in general graph: NP-hard (no known poly algorithm)
  
  SHORTEST path in DAG: O(V + E) — DAG DP
  LONGEST  path in DAG: O(V + E) — DAG DP (same algorithm, just max!)
  
  WHY? In a DAG, no cycles means:
    → Finite number of paths (no infinite loops)
    → Topological order exists → valid DP evaluation order
    → Both min and max are equally easy
    
  In a cyclic graph:
    → Shortest path: cycles can't help (adding a positive cycle
      makes path longer, so Dijkstra/BF can ignore them)
    → Longest path: cycles CAN "help" — go around a positive cycle 
      infinitely → longest path = infinity → problem is ill-defined
      → Must restrict to SIMPLE paths → NP-hard
      
  This is THE fundamental reason DAGs are so important in CS:
    "DAGs make hard problems easy."

from collections import defaultdict, deque

def min_semesters(n, prerequisites):
    """
    LeetCode 1136: Parallel Courses
    
    Given N courses and prerequisites, find the MINIMUM number of 
    semesters to take ALL courses. You can take any number of courses 
    per semester, as long as prerequisites are met.
    
    This is EXACTLY longest path in DAG!
    → Each course takes 1 semester (duration = 1 for all)
    → Longest chain of prerequisites = minimum semesters
    
    If cycle exists → impossible → return -1
    
    Connects to:
      - Problem 21 (Course Schedule I — can we finish?)
      - Problem 22 (Course Schedule II — what order?)
      - This problem: how FAST can we finish?
    """
    adj = defaultdict(list)
    in_degree = [0] * (n + 1)  # 1-indexed courses
    
    for pre, course in prerequisites:
        adj[pre].append(course)
        in_degree[course] += 1
    
    # dp[course] = minimum semester in which this course finishes
    dp = [0] * (n + 1)
    
    queue = deque()
    for course in range(1, n + 1):
        if in_degree[course] == 0:
            queue.append(course)
            dp[course] = 0  # can start in semester 0 (no prerequisites)
    
    processed = 0
    
    while queue:
        course = queue.popleft()
        processed += 1
        
        # This course takes 1 semester
        dp[course] += 1
        
        for next_course in adj[course]:
            dp[next_course] = max(dp[next_course], dp[course])
            in_degree[next_course] -= 1
            if in_degree[next_course] == 0:
                queue.append(next_course)
    
    # Cycle check
    if processed != n:
        return -1  # cycle → impossible
    
    return max(dp[1:n+1])

from collections import defaultdict, deque

def critical_path_analysis(n, edges, durations):
    """
    Complete Critical Path Analysis:
    
    For each task, compute:
      - Earliest Start (ES): earliest time task can begin
      - Earliest Finish (EF): ES + duration
      - Latest Start (LS): latest time task can begin without delaying project
      - Latest Finish (LF): latest time task can finish without delaying project
      - Slack (Float): LS - ES = LF - EF
        → Slack = 0 means task is CRITICAL (on critical path)
        → Slack > 0 means task has flexibility
    
    Two-pass approach:
      Forward pass: compute ES and EF (longest path, as before)
      Backward pass: compute LF and LS (reverse longest path)
    
    This mirrors the REROOTING idea from Problem 46:
      Forward pass = bottom-up
      Backward pass = top-down
    """
    # ─── Build forward and reverse graphs ───
    adj_fwd = defaultdict(list)      # forward edges
    adj_rev = defaultdict(list)      # reverse edges
    in_degree_fwd = [0] * n
    in_degree_rev = [0] * n          # out-degree in original = in-degree in reverse
    
    for u, v in edges:
        adj_fwd[u].append(v)
        adj_rev[v].append(u)
        in_degree_fwd[v] += 1
        in_degree_rev[u] += 1
    
    # ─── FORWARD PASS: Compute ES and EF ───
    ES = [0] * n  # earliest start
    EF = [0] * n  # earliest finish
    
    queue = deque()
    for node in range(n):
        if in_degree_fwd[node] == 0:
            queue.append(node)
    
    fwd_in = in_degree_fwd[:]
    while queue:
        node = queue.popleft()
        EF[node] = ES[node] + durations[node]
        
        for nei in adj_fwd[node]:
            ES[nei] = max(ES[nei], EF[node])
            fwd_in[nei] -= 1
            if fwd_in[nei] == 0:
                queue.append(nei)
    
    # ─── Project completion time ───
    project_duration = max(EF)
    
    # ─── BACKWARD PASS: Compute LF and LS ───
    LF = [project_duration] * n  # latest finish (start with project end)
    LS = [0] * n                  # latest start
    
    queue = deque()
    for node in range(n):
        if in_degree_rev[node] == 0:  # sink nodes (no outgoing edges)
            queue.append(node)
    
    rev_in = in_degree_rev[:]
    while queue:
        node = queue.popleft()
        LS[node] = LF[node] - durations[node]
        
        for pred in adj_rev[node]:
            LF[pred] = min(LF[pred], LS[node])
            rev_in[pred] -= 1
            if rev_in[pred] == 0:
                queue.append(pred)
    
    # ─── Compute Slack and identify critical path ───
    slack = [LS[i] - ES[i] for i in range(n)]
    critical = [i for i in range(n) if slack[i] == 0]
    
    # ─── Print results ───
    print(f"Project Duration: {project_duration} days\n")
    print(f"{'Task':<6}{'Dur':<5}{'ES':<5}{'EF':<5}{'LS':<5}{'LF':<5}{'Slack':<7}{'Critical'}")
    print("─" * 48)
    for i in range(n):
        is_crit = "★" if slack[i] == 0 else ""
        print(f"{i:<6}{durations[i]:<5}{ES[i]:<5}{EF[i]:<5}{LS[i]:<5}{LF[i]:<5}{slack[i]:<7}{is_crit}")
    
    print(f"\nCritical Path Tasks: {critical}")
    
    return project_duration, critical, slack

# ═══════════════════════════════════════════════════════════════
#  UNIVERSAL DAG DP TEMPLATE
# ═══════════════════════════════════════════════════════════════

from collections import defaultdict, deque

def dag_dp_template(n, edges, node_values, combine_fn, init_val, base_val):
    """
    Generic DAG DP solver.
    
    Parameters:
        n:           number of nodes
        edges:       directed edges [u, v] (or [u, v, w] for weighted)
        node_values: value at each node (or None for edge-weighted)
        combine_fn:  how to combine (max for longest, min for shortest, 
                     sum for counting)
        init_val:    initial dp value (0 for longest, inf for shortest)
        base_val:    base dp value for source nodes
    
    Returns:
        dp: array where dp[i] = optimal value for node i
    """
    adj = defaultdict(list)
    in_degree = [0] * n
    
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
    
    dp = [init_val] * n
    
    queue = deque()
    for node in range(n):
        if in_degree[node] == 0:
            dp[node] = base_val
            queue.append(node)
    
    while queue:
        node = queue.popleft()
        
        # Add node value if node-weighted
        if node_values is not None:
            dp[node] += node_values[node]
        
        for nei in adj[node]:
            dp[nei] = combine_fn(dp[nei], dp[node])
            in_degree[nei] -= 1
            if in_degree[nei] == 0:
                queue.append(nei)
    
    return dp
