Problem 35: Articulation Points (Cut Vertices)

[Algorithm Family: Tarjan's Algorithm — Articulation Point Finding]

[Phase 4: Advanced — "I can handle complex multi-concept problems"]

PROBLEM STATEMENT

Same network as Problem 34, but now the question changes:

Instead of finding critical CABLES (bridges/edges),
find critical SERVERS (articulation points/nodes).

An "articulation point" (or "cut vertex") is a server whose
REMOVAL (along with all its cables) would disconnect the network.

Input:
  n = 7  (servers: 0, 1, 2, 3, 4, 5, 6)
  edges = [[0,1], [1,2], [2,0], [1,3], [3,4], [3,5], [5,6]]

Output: [1, 3]

Explanation:
  Remove server 1 (and all its cables):
    → {0,2} can no longer reach {3,4,5,6} → DISCONNECTED
    → Server 1 is an articulation point

  Remove server 3 (and all its cables):
    → {4} isolated, {5,6} isolated from {0,1,2} → DISCONNECTED
    → Server 3 is an articulation point

  No other server removal disconnects the network.

STEP 2: BRIDGE vs ARTICULATION POINT — The Difference

  ╔═══════════════════════════════════════════════════════════════════╗
  ║                                                                   ║
  ║  BRIDGE (Problem 34):                                             ║
  ║    → Remove an EDGE → graph disconnects?                         ║
  ║    → "Which cables are critical?"                                ║
  ║    → Condition: low[v] > disc[u]                                 ║
  ║                 (strictly greater)                                ║
  ║                                                                   ║
  ║  ARTICULATION POINT (Problem 35):                                 ║
  ║    → Remove a NODE (+ all its edges) → graph disconnects?        ║
  ║    → "Which servers are critical?"                               ║
  ║    → Condition: low[v] >= disc[u]                                ║
  ║                 (greater or EQUAL)                                ║
  ║    → PLUS special case for root of DFS tree                      ║
  ║                                                                   ║
  ║  WHY >= instead of >?                                             ║
  ║                                                                   ║
  ║    Bridge (>):                                                    ║
  ║      "v's subtree can't reach u AT ALL"                          ║
  ║      → Removing EDGE (u,v) disconnects v's subtree               ║
  ║      → Even if v reaches u through back edge, the edge survives  ║
  ║                                                                   ║
  ║    Artic point (>=):                                              ║
  ║      "v's subtree can't reach ABOVE u"                           ║
  ║      → Even if v reaches u (low[v] == disc[u]),                  ║
  ║        when we REMOVE NODE u, that path dies too!                ║
  ║      → v's subtree is stranded                                   ║
  ║      → So u is an articulation point                             ║
  ║                                                                   ║
  ║  VISUAL:                                                          ║
  ║                                                                   ║
  ║    Bridge case:           Artic point case:                       ║
  ║                                                                   ║
  ║       u                      u                                    ║
  ║       │                      │                                    ║
  ║       v (low[v] = disc[u])   v (low[v] = disc[u])                ║
  ║       │                      │                                    ║
  ║       └──back to u           └──back to u                        ║
  ║                                                                   ║
  ║    Remove edge u─v:          Remove NODE u:                       ║
  ║    v→back→u still works!     v→back→u... u is GONE!              ║
  ║    → NOT bridge              → v is stranded                      ║
  ║                              → u IS artic point                   ║
  ║                                                                   ║
  ╚═══════════════════════════════════════════════════════════════════╝

STEP 3: BRUTE FORCE FIRST

Naive Approach: Remove Each Node, Check Connectivity

  For each node u:
    1. Remove node u and all its edges
    2. Run BFS/DFS on remaining graph to check connectivity (Problem 5)
    3. If remaining graph (excluding u) is disconnected → u is artic point
    4. Put node u back

  Time per node: O(V + E) for connectivity check
  Total: V × O(V + E) = O(V × (V + E))

  For V = 100,000, E = 200,000:
    → 100,000 × 300,000 = 3 × 10¹⁰ → TOO SLOW

  We need O(V + E) — single DFS pass, like Tarjan's for bridges.

STEP 4: IDENTIFYING THE GRAPH PATTERN

The Two Cases for Articulation Points

  ╔═══════════════════════════════════════════════════════════════════╗
  ║                                                                   ║
  ║  A node u is an ARTICULATION POINT if EITHER:                    ║
  ║                                                                   ║
  ║  CASE 1: u is the ROOT of DFS tree AND has 2+ children          ║
  ║                                                                   ║
  ║    WHY? If root has 2+ children in DFS tree:                     ║
  ║      → Those children are in separate subtrees                   ║
  ║      → The ONLY path between them goes through root              ║
  ║      → Remove root → children's subtrees disconnected            ║
  ║                                                                   ║
  ║    If root has only 1 child:                                      ║
  ║      → Removing root just removes one "layer"                    ║
  ║      → Remaining graph is still connected (the child's subtree)  ║
  ║                                                                   ║
  ║    Example:                                                       ║
  ║      Root with 2 children:     Root with 1 child:                ║
  ║           R                          R                           ║
  ║          / \                         |                           ║
  ║         A   B                        A                           ║
  ║        / \   \                      / \                          ║
  ║       C   D   E                    B   C                         ║
  ║                                                                   ║
  ║      Remove R → {A,C,D} and       Remove R → {A,B,C} still      ║
  ║      {B,E} disconnected!          connected → NOT artic point    ║
  ║                                                                   ║
  ║                                                                   ║
  ║  CASE 2: u is NOT root AND has child v where low[v] >= disc[u]  ║
  ║                                                                   ║
  ║    WHY? v's subtree can only reach as far back as u.             ║
  ║      → Remove u → v's subtree can't reach u's ancestors          ║
  ║      → v's subtree is disconnected from the rest                 ║
  ║                                                                   ║
  ║    Note: We need this for AT LEAST ONE child v.                  ║
  ║    (Even if one child can escape, another might not)             ║
  ║                                                                   ║
  ╚═══════════════════════════════════════════════════════════════════╝

STEP 6: WHY ROOT IS SPECIAL — Deep Intuition

  ╔═══════════════════════════════════════════════════════════════════╗
  ║  WHY DOES THE ROOT NEED A DIFFERENT RULE?                        ║
  ╠═══════════════════════════════════════════════════════════════════╣
  ║                                                                   ║
  ║  For NON-ROOT node u:                                             ║
  ║    → u has a PARENT in DFS tree                                   ║
  ║    → If child v can't reach above u → removing u disconnects     ║
  ║      v from u's parent → graph breaks                             ║
  ║    → Condition: low[v] >= disc[u] works perfectly                ║
  ║                                                                   ║
  ║  For ROOT node:                                                   ║
  ║    → Root has NO parent                                           ║
  ║    → There's nothing "above" the root to disconnect from          ║
  ║    → low[v] >= disc[root] is ALWAYS true for any child v         ║
  ║      (disc[root] = 0, smallest possible)                         ║
  ║    → So the >= condition would mark ALL roots as artic points!   ║
  ║    → That's WRONG — root might not be an artic point              ║
  ║                                                                   ║
  ║  The REAL test for root:                                          ║
  ║    → If root has 2+ DFS children:                                ║
  ║      → Those subtrees are independent (no cross edges in DFS)    ║

Primary Solution: Tarjan's Articulation Points

from collections import defaultdict

def find_articulation_points(n, edges):
    """
    Tarjan's Algorithm — Find all articulation points in an undirected graph.
    
    An articulation point is a node whose removal disconnects the graph.
    
    Two conditions (from Problem 34's disc/low framework):
    
      CASE 1 — ROOT:
        DFS root is artic point if it has >= 2 DFS children.
        
      CASE 2 — NON-ROOT:
        Non-root u is artic point if ANY child v has low[v] >= disc[u].
        "v's subtree can't reach above u → removing u disconnects v"
    
    Key difference from bridges (Problem 34):
      Bridge:     low[v] >  disc[u]  (strictly greater)
      Artic pt:   low[v] >= disc[u]  (greater or EQUAL)
      + root special case
    
    Args:
        n: number of vertices (0 to n-1)
        edges: list of [u, v] (undirected edges)
    
    Returns:
        list of articulation points
    
    Time: O(V + E) — single DFS pass
    Space: O(V + E)
    """
    # ─── Build adjacency list ───
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    # ─── Initialize arrays ───
    disc = [-1] * n
    low = [-1] * n
    is_artic = [False] * n       # use boolean array to avoid duplicates
    timer = [0]
    
    def dfs(u, parent):
        # Assign discovery time and initial low value
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        
        children_count = 0       # count of DFS tree children (for root check)
        
        for v in graph[u]:
            if disc[v] == -1:
                # ─── Tree edge: v is unvisited ───
                children_count += 1
                dfs(v, u)
                
                # Update low from child
                low[u] = min(low[u], low[v])
                
                # ★ CASE 1: Root check ★
                # Root with 2+ children is articulation point
                if parent == -1 and children_count >= 2:
                    is_artic[u] = True
                
                # ★ CASE 2: Non-root check ★
                # If child v can't reach above u → u is artic point
                if parent != -1 and low[v] >= disc[u]:
                    is_artic[u] = True
            
            elif v != parent:
                # ─── Back edge: v is visited and not parent ───
                low[u] = min(low[u], disc[v])
    
    # ─── Run DFS from every unvisited node ───
    for i in range(n):
        if disc[i] == -1:
            dfs(i, -1)
    
    # ─── Collect results ───
    return [i for i in range(n) if is_artic[i]]

from collections import defaultdict

def find_articulation_points_iterative(n, edges):
    """
    Iterative version to avoid Python recursion limit.
    Same logic as recursive, just manually managing the stack.
    """
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    disc = [-1] * n
    low = [-1] * n
    parent = [-1] * n
    is_artic = [False] * n
    children_count = [0] * n
    timer = 0
    
    for start in range(n):
        if disc[start] != -1:
            continue
        
        stack = [(start, iter(graph[start]))]
        disc[start] = low[start] = timer
        timer += 1
        
        while stack:
            u, neighbors = stack[-1]
            
            try:
                v = next(neighbors)
                
                if disc[v] == -1:
                    # Tree edge → go deeper
                    children_count[u] += 1
                    parent[v] = u
                    disc[v] = low[v] = timer
                    timer += 1
                    stack.append((v, iter(graph[v])))
                
                elif v != parent[u]:
                    # Back edge → update low
                    low[u] = min(low[u], disc[v])
            
            except StopIteration:
                # All neighbors processed → backtrack
                stack.pop()
                
                if stack:
                    p = stack[-1][0]   # parent node
                    low[p] = min(low[p], low[u])
                    
                    # Articulation point checks
                    # Case 1: Root
                    if parent[p] == -1 and children_count[p] >= 2:
                        is_artic[p] = True
                    # Case 2: Non-root
                    if parent[p] != -1 and low[u] >= disc[p]:
                        is_artic[p] = True
                    
                    u = p   # restore u for next iteration
    
    return [i for i in range(n) if is_artic[i]]

from collections import defaultdict

def find_bridges_and_artic_points(n, edges):
    """
    Find BOTH bridges and articulation points in a single DFS.
    
    Combines Problems 34 and 35.
    Same DFS, just check both conditions at each step.
    Still O(V + E) — no extra cost!
    """
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    disc = [-1] * n
    low = [-1] * n
    is_artic = [False] * n
    bridges = []
    timer = [0]
    
    def dfs(u, parent):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        children_count = 0
        
        for v in graph[u]:
            if disc[v] == -1:
                children_count += 1
                dfs(v, u)
                low[u] = min(low[u], low[v])
                
                # Bridge check (Problem 34): strictly greater
                if low[v] > disc[u]:
                    bridges.append([u, v])
                
                # Artic point check (Problem 35)
                # Root case
                if parent == -1 and children_count >= 2:
                    is_artic[u] = True
                # Non-root case: greater or equal
                if parent != -1 and low[v] >= disc[u]:
                    is_artic[u] = True
            
            elif v != parent:
                low[u] = min(low[u], disc[v])
    
    for i in range(n):
        if disc[i] == -1:
            dfs(i, -1)
    
    artic_points = [i for i in range(n) if is_artic[i]]
    return bridges, artic_points

STEP 13: WHAT IF THE GRAPH WERE...

What if Directed?

  Articulation points are defined for UNDIRECTED graphs.
  
  For directed graphs:
    → "Which node removal disconnects the graph?"
      depends on definition of "connected" (weakly vs strongly)
    → For strong connectivity:
      → Related to Strongly Connected Components (Problem 36)
      → A node whose removal changes the SCC structure
    → Much more complex analysis

What if Disconnected?

  Our code handles this with the outer loop:
    for i in range(n):
        if disc[i] == -1:
            dfs(i, -1)
  
  → Each connected component processed independently
  → Artic points found within each component
  → Note: removing an artic point from one component
    doesn't affect other components

What if We Want Biconnected Components?

  A biconnected component is a maximal subgraph with NO artic points.
  → Removing any single node from it keeps it connected.
  → The graph can be decomposed into biconnected components
    separated by articulation points.
  
  Our graph:
    Biconnected component 1: {0, 1, 2}  (triangle)
    Biconnected component 2: {1, 3}      (bridge)
    Biconnected component 3: {3, 4}      (bridge)
    Biconnected component 4: {3, 5}      (bridge)
    Biconnected component 5: {5, 6}      (bridge)
  
  Note: artic points belong to MULTIPLE biconnected components.
  This is an advanced extension beyond standard interviews.

  │  2. "Is this different from bridges?"                              │
  │      → YES — bridges = critical EDGES, artic points = critical    │

STEP 18: KEY TAKEAWAY

  ╔════════════════════════════════════════════════════════════════════╗
  ║                                                                    ║
  ║  NEW CONCEPT: ARTICULATION POINTS (Cut Vertices)                   ║
  ║                                                                    ║
  ║  SAME framework as bridges (Problem 34):                           ║
  ║    → disc[] = discovery time                                       ║
  ║    → low[]  = earliest ancestor reachable from subtree            ║
  ║    → Single DFS pass, O(V + E)                                    ║
  ║                                                                    ║
  ║  DIFFERENT conditions:                                             ║
  ║                                                                    ║
  ║    ┌─────────────────────────────────────────────────────────────┐ ║
  ║    │                                                             │ ║
  ║    │  BRIDGES (P34):                                             │ ║
  ║    │    low[v] > disc[u]  (strictly greater)                     │ ║
  ║    │    No special root case                                     │ ║
  ║    │    Result: edges                                            │ ║
  ║    │                                                             │ ║
  ║    │  ARTICULATION POINTS (P35):                                 │ ║
  ║    │    Non-root: low[v] >= disc[u]  (greater or EQUAL)         │ ║
  ║    │    Root: children_count >= 2     (special case)             │ ║
  ║    │    Result: nodes                                            │ ║
  ║    │                                                             │ ║
  ║    └─────────────────────────────────────────────────────────────┘ ║
  ║                                                                    ║
  ║  WHY >= instead of >:                                              ║
  ║    → Removing a NODE kills ALL edges through it                   ║
  ║    → Even if subtree reaches u exactly (low = disc[u]),           ║
  ║      removing u destroys that path → subtree is stranded         ║
  ║    → For bridges, removing an EDGE doesn't kill the endpoint     ║
  ║      node, so reaching u still works → need strict >             ║
  ║                                                                    ║
  ║  WHY root is special:                                              ║
  ║    → Root has no parent → >= condition is always true             ║
  ║    → Instead check: does root have 2+ independent subtrees?      ║
  ║    → If yes → removing root disconnects them → artic point       ║
  ║                                                                    ║
  ║  CAN COMBINE WITH P34:                                            ║
  ║    → One DFS finds BOTH bridges AND artic points                  ║
  ║    → Check both conditions at each tree edge                      ║
  ║    → Still O(V + E)                                               ║
  ║                                                                    ║
  ╚════════════════════════════════════════════════════════════════════╝

from collections import defaultdict

def find_articulation_points(n, edges):
    """
    Tarjan's articulation point finding. O(V+E) time.
    Root: 2+ DFS children. Non-root: low[v] >= disc[u].
    """
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    disc = [-1] * n
    low = [-1] * n
    is_artic = [False] * n
    timer = [0]
    
    def dfs(u, parent):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        children = 0
        
        for v in graph[u]:
            if disc[v] == -1:
                children += 1
                dfs(v, u)
                low[u] = min(low[u], low[v])
                
                # Root: 2+ children → artic point
                if parent == -1 and children >= 2:
                    is_artic[u] = True
                # Non-root: child can't reach above me → artic point
                if parent != -1 and low[v] >= disc[u]:
                    is_artic[u] = True
            
            elif v != parent:
                low[u] = min(low[u], disc[v])
    
    for i in range(n):
        if disc[i] == -1:
            dfs(i, -1)
    
    return [i for i in range(n) if is_artic[i]]
